전체 데이터 백업

Supabase의 주요 테이블(논문·시간표·취업·관계도)을 한 번에 받아 복원용 SQL 또는 JSON으로 저장합니다. 받은 파일을 GitHub 리포(KIT_sodi)의 backup/ 폴더에 올려 커밋하면 백업이 됩니다.

⚠ KIT_sodi는 공개 리포입니다. 학생 실명이 들어간 파일을 그대로 커밋하면 이름이 공개로 노출됩니다. 공개 리포에 올릴 백업은 위 이름 마스킹을 켠 채로 받으세요. 실명이 포함된 전체 백업이 필요하면 마스킹을 끄고 받아 개인 PC나 비공개 저장소에만 보관하세요.

복원 방법

받은 복원용 SQL을 Supabase의 SQL Editor에 붙여넣고 Run 하면, 각 테이블을 비우고 백업 시점 데이터로 되돌립니다. (스키마(테이블 구조)는 schema_cfms.sql로 먼저 만들고, 권한 정책은 enable_rls_cfms.sql로 적용합니다.)
